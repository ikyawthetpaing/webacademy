Welcome, TypeScript enthusiasts! This chapter delves into the world of enums, a powerful tool for defining sets of named constants in your code. Enums enhance code readability, maintainability, and type safety by providing a clear definition for constant values. Buckle up and get ready to explore the exciting world of enums in TypeScript!

## 1. The Enum Advantage: Organizing Constants with Style

Imagine a cluttered toolbox with various tools unnamed – enums in TypeScript function similarly to labeled compartments in a toolbox. They allow you to define a set of named constants, making your code more readable and easier to maintain.

**Defining Enums:**

The `enum` keyword is used to define enums in TypeScript:

```typescript
enum Weekdays {
  Monday,
  Tuesday,
  Wednesday,
  Thursday,
  Friday,
  Saturday,
  Sunday,
}

let today: Weekdays = Weekdays.Wednesday;
console.log(`Today is: ${today}`); // Output: Today is: 2 (underlying numeric value)

// Accessing enum values by name
let weekendDay = Weekdays.Saturday;
console.log(`Weekend is approaching! It's ${weekendDay}`); // Output: Weekend is approaching! It's 5
```

**Remember:**

- Enums provide a clear and concise way to define and manage sets of related constants.
- By using named constants instead of magic numbers, your code becomes more self-documenting and easier to understand.

## 2. Unveiling Flexibility: Enums with String Values

Imagine a toolbox with compartments labeled not just with numbers but with descriptive names – enums in TypeScript can also have string values:

```typescript
enum PaymentStatus {
  Pending = "payment-pending",
  Succeeded = "payment-succeeded",
  Failed = "payment-failed",
}

let payment1Status = PaymentStatus.Succeeded;
console.log(`Payment status: ${payment1Status}`); // Output: Payment status: payment-succeeded

let payment2Status = PaymentStatus.Failed;
console.log(`Payment failed! Status: ${PaymentStatus[payment2Status]}`); // Accessing string value by numeric value
// Output: Payment failed! Status: payment-failed
```

**Remember:**

- Enums can be defined with string values, providing more descriptive labels for your constants.
- You can access both the numeric value (by default) and the corresponding string value using the enum name as an index.

## 3. Bitwise Magic: Enums as Bit Flags

Imagine a toolbox with compartments that can represent multiple flags (e.g., open/closed, locked/unlocked) – enums in TypeScript can be used as bit flags:

```typescript
enum Permissions {
  Read = 1,
  Write = 2,
  Execute = 4,
  All = Read | Write | Execute, // Combining bit flags using bitwise OR operator
}

let userPermissions = Permissions.Read | Permissions.Write;
console.log(`User permissions: ${userPermissions}`); // Output: 3 (combined numeric value)

// Checking if a specific flag is enabled using bitwise AND operator
let hasReadPermission = (userPermissions & Permissions.Read) !== 0;
console.log(`Has read permission: ${hasReadPermission}`); // Output: Has read permission: true
```

**Remember:**

- Enums can be used as bit flags, allowing you to represent combinations of permissions or options using bitwise operations.
- This is particularly useful for scenarios where multiple flags can be combined.

## 4. Unveiling the Mystery: Underlying Numeric Values (Optional)

**Optional Topic:**

By default, enums have an underlying numeric value starting from 0. You can optionally assign specific numeric values to enum members:

```typescript
enum HttpStatus {
  OK = 200,
  BadRequest = 400,
  NotFound = 404,
}

let successfulResponse = HttpStatus.OK;
console.log(`Status code: ${successfulResponse}`); // Output: Status code: 200
```

**Remember:**

- Understanding the underlying numeric values of enums can be helpful in certain scenarios, especially when working with APIs or libraries that use numeric codes.
- However, it's generally recommended to rely on the named constants for better readability and maintainability.

## 5. Code Example: Utilizing Enums (Task Management)

Let's build a simple task management example to illustrate enums in action:

```typescript
enum TaskStatus {
  Open = "open",
  InProgress = "in-progress",
  Completed = "completed",
}

class Task {
  title: string;
  status: TaskStatus;

  constructor(title: string, status: TaskStatus = TaskStatus.Open) {
    // Default status (optional)
    this.title = title;
    this.status = status;
  }

  markInProgress(): void {
    this.status = TaskStatus.InProgress;
  }

  markCompleted(): void {
    this.status = TaskStatus.Completed;
  }

  printTaskDetails(): void {
    console.log(`Task: ${this.title}, Status: ${this.status}`);
  }
}

let task1 = new Task("Write a blog post");
task1.printTaskDetails(); // Output: Task: Write a blog post, Status: open

task1.markInProgress();
task1.printTaskDetails(); // Output: Task: Write a blog post, Status: in-progress

task1.markCompleted();
task1.printTaskDetails(); // Output: Task: Write a blog post, Status: completed

let task2 = new Task("Review code", TaskStatus.InProgress); // Setting initial status
task2.printTaskDetails(); // Output: Task: Review code, Status: in-progress
```

**Explanation:**

1. We define a `TaskStatus` enum with string values for different task states.
2. The `Task` class has a `status` property of type `TaskStatus`, promoting type safety.
3. We create tasks with different initial statuses and demonstrate methods to change the status using the enum values.

**Remember:**

- Enums enhance code readability, maintainability, and type safety by providing a clear way to define and manage sets of related constants.
